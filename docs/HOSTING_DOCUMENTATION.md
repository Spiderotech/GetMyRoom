# Hosting Documentation

## Current Hosting Status

## Deployment Components

The production system should include:

| Component | Recommended Service | Notes |
| --- | --- | --- |
| Frontend | Vercel, Netlify, Cloudflare Pages, or S3 + CloudFront | Build output is generated by Vite in `FRONTEND/dist/` |
| Backend | Render, Railway, Fly.io, AWS EC2, ECS, or Elastic Beanstalk | Runs Node.js/Express API |
| Database | MongoDB Atlas | Use separate development and production clusters or databases |
| Object Storage | AWS S3 | Used for property images and file upload pre-signed URLs |
| DNS | Domain registrar or Cloudflare | Point domain/subdomains to frontend and API |
| SSL | Hosting provider or AWS Certificate Manager | Required for production |
| Email | Gmail app password, SMTP provider, SES, SendGrid, or Mailgun | Used for property inquiry emails |

## S3 Plan

Current status:

```text
S3 bucket: not created for this environment
S3 credentials: not created for this environment
```

Recommended bucket setup:

```text
Development bucket: getmyroom-dev-assets
Production bucket: getmyroom-prod-assets
Region: ap-south-1 or the region closest to the app users
Public access: keep blocked unless using CloudFront or explicit object URLs
Upload method: backend-generated pre-signed PUT URLs
```

Recommended S3 environment variables:

```env
AWS_REGION=ap-south-1
S3_BUCKET_NAME=getmyroom-dev-assets
S3_ACESS_KEY=replace_with_dev_access_key
S3_SECRET_KEY=replace_with_dev_secret_key
```

Production should use separate credentials:

```env
AWS_REGION=ap-south-1
S3_BUCKET_NAME=getmyroom-prod-assets
S3_ACESS_KEY=replace_with_prod_access_key
S3_SECRET_KEY=replace_with_prod_secret_key
```

Recommended IAM permissions:

```json
{
  "Version": "2012-10-17",
  "Statement": [
    {
      "Effect": "Allow",
      "Action": [
        "s3:PutObject",
        "s3:GetObject"
      ],
      "Resource": "arn:aws:s3:::getmyroom-prod-assets/*"
    }
  ]
}
```

## Backend Hosting Steps

1. Create the backend app on the hosting platform.
2. Connect the git repository.
3. Set the root directory to `BACKEND` if the platform supports monorepo root selection.
4. Set install command:

```bash
npm install
```

5. Set start command:

```bash
npm start
```

6. Add backend environment variables.
7. Deploy the service.
8. Confirm health by opening:

```text
https://api-domain.example.com/api/v1/property/getallproperty
```

The current backend has no dedicated health check route. Add one before production:

```text
GET /api/v1/health
Response: { "status": "ok" }
```

## Frontend Hosting Steps

1. Create the frontend app on the hosting platform.
2. Connect the git repository.
3. Set the root directory to `FRONTEND`.
4. Set install command:

```bash
npm install
```

5. Set build command:

```bash
npm run build
```

6. Set output directory:

```text
dist
```

7. Configure frontend environment variables.
8. Deploy.
9. Configure SPA fallback so all React routes serve `index.html`.

## Environment Variables by Environment

Development:

```env
PORT=3000
MONGO_URI=mongodb://localhost:27017/getmyroom
ACESS_TOKEN_SCERET=dev_access_secret
REFRESH_TOKEN_SECRET=dev_refresh_secret
AWS_REGION=ap-south-1
S3_BUCKET_NAME=getmyroom-dev-assets
S3_ACESS_KEY=dev_key_after_bucket_creation
S3_SECRET_KEY=dev_secret_after_bucket_creation
EMAIL=dev_sender@example.com
PASSWORD=dev_email_password
```

Production:

```env
PORT=3000
MONGO_URI=mongodb+srv://<user>:<password>@<cluster>/<database>
ACESS_TOKEN_SCERET=strong_production_access_secret
REFRESH_TOKEN_SECRET=strong_production_refresh_secret
AWS_REGION=ap-south-1
S3_BUCKET_NAME=getmyroom-prod-assets
S3_ACESS_KEY=prod_key_after_bucket_creation
S3_SECRET_KEY=prod_secret_after_bucket_creation
EMAIL=production_sender@example.com
PASSWORD=production_email_or_smtp_password
```

Frontend variables should be added after refactoring constants:

```env
VITE_API_BASE_URL=http://localhost:3000/api/v1
```

Production:

```env
VITE_API_BASE_URL=https://api.getmyroom.co.uk/api/v1
```
